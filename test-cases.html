Test Cases: OpenCart Framework

Test Suite: Cart Quantity Boundaries & Shipping Rules
Author: Gururaj Hegde
Feature: OpenCart Core Cart & Checkout (v4.0.2.3)
Setup notes: Ensure the local OpenCart database is populated with the default test data. Make sure the product "MacBook" (Product ID: 43) has exactly 500 items in stock for testing the max-stock boundary.

1. Boundary & Extreme Value Testing (Add to Cart Quantity)

ID Test Scenario Steps Expected Result Status Notes
TC-01 Minimum invalid quantity (0) 1. Go to MacBook product page.
2. Enter 0 in Qty.
3. Click Add to Cart.		 Error message displays: "Quantity must be greater than 0." Button fails gracefully.
TC-02 Minimum valid quantity (1) 1. Enter 1 in Qty.
2. Click Add to Cart.		 Success banner appears. Mini-cart updates to 1 item.
TC-03 Exact maximum stock limit (500) 1. Enter 500 in Qty.
2. Click Add to Cart.		 Success banner appears. Cart allows the exact stock amount. Requires clearing the cart before this step.
TC-04 Over maximum stock limit (501) 1. Enter 501 in Qty.
2. Click Add to Cart.		 Item is added, but Shopping Cart view shows a warning: "Products marked with *** are not available in the desired quantity or not in stock!" OpenCart allows adding out-of-stock items by default, but blocks checkout if "Stock Checkout" is disabled in Admin.
TC-05 Negative quantity input (-5) 1. Type -5 in Qty.
2. Click Add to Cart.		 Front-end validation strips the minus sign or returns an error. Should not allow negative numbers.
TC-06 Huge integer overflow (999999999) 1. Enter 999999999 in Qty.
2. Click Add to Cart.		 Handled gracefully. Cart should cap at max stock or throw a standard validation error, not a DB INT overflow error.
