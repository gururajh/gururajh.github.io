Ticket ID: OC-4192

Project: OpenCart Core - Checkout Module
Logged by: Gururaj Hegde
Date: 23.03.2026
Category: Cart / Payment Gateway
Severity: High
Priority: P1

Summary

Cart throws an unhandled server exception (HTTP 500) when a Flat Rate coupon exceeds the total cart value.

Description

Problem:
When a customer applies a valid coupon code configured as a "Fixed Amount" discount, and that discount is strictly greater than the subtotal of the items in the cart, the application crashes on the backend.

The expected behavior (as seen in older versions like v3.x) is that the cart total should simply be reduced to $0.00, or the UI should display a warning like: "Warning: Coupon discount exceeds cart total!" Currently, it hits an unhandled PHP exception during the total calculation.

Steps to reproduce:

  1. Log into the OpenCart Admin panel and create a Coupon:
    • Name: Promo50
    • Type: Fixed Amount
    • Discount: $50.00
  2. Navigate to the storefront (e.g., demo.opencart.com local instance).
  3. Add a low-priced item to the cart (e.g., "Apple Cinema 30", configured to $20.00).
  4. Navigate to the Shopping Cart view (/index.php?route=checkout/cart).
  5. Open the "Use Coupon Code" accordion tab.
  6. Enter Promo50 and click "Apply Coupon".
  7. Result: The AJAX request to index.php?route=extension/opencart/total/coupon|save returns a 500 Internal Server Error. The front-end hangs with a loading spinner indefinitely.

Environment:
- Application: OpenCart v4.0.2.3 (Default Theme)
- Server: PHP 8.2.10, Nginx, MySQL 8.0
- Browser: Chrome 122.0 (Windows 11) & Safari 17.3 (macOS Sonoma)

Ticket ID: OC-4205

Project: OpenCart Core - Customer Account
Logged by: Gururaj Hegde
Date: 23.03.2026
Category: UI / Form Validation
Severity: Minor
Priority: P3

Summary

"Telephone" field in the Registration form allows alphabet characters without triggering front-end validation.

Description

Problem:
The telephone input field during account registration (/index.php?route=account/register) does not restrict alpha characters (A-Z). While the database ultimately saves the string, allowing letters in a phone number field can break third-party SMS gateway integrations (like Twilio) downstream.

Steps to reproduce:

  1. Go to the "Register Account" page.
  2. Fill out the mandatory fields (First Name, Last Name, Email, Password).
  3. In the "Telephone" field, enter +1-800-CALL-NOW-ABC.
  4. Agree to the Privacy Policy and click "Continue".
  5. Result: The account is successfully created without any warning. The Telephone number is saved including the letters exactly as typed.

Environment:
- Application: OpenCart v4.0.2.3 (Default Theme)
- Browser: Firefox 123.0 (Desktop)
